  - `jvmPomTask.map { it.destination }` → `.flatMap { it.destination }` (lambda now returns a provider —
    avoids `Provider<RegularFileProperty>`).
  - `destination.readText()`, `docBuilder.parse(destination)`, `StreamResult(destination)`,
    `jvmPom.get()` → resolved with `.get().asFile` inside the `doLast` task action (a plain `File` is
    required by the JDK XML / `readText` APIs).

  plugin and its `develocity { }` block were removed. This is a third-party plugin/config relaxation, not
  a Provider API change, and should be restored when migrating against a released Gradle 10.
